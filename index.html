Describe how your dashboard answers the questions presented. You don't have to address every question directly, but should at a high level address the main questions.

The first question asks to display the top 10 games of all time or a specified year. The result is shown using a bar plot. The bar plot is fit for this question as it displays total sales for each game eloquently. Moreover, we can compare two or more games using the current bar plot. Irrespective of the fact that the data is sorted or not, a bar plot can clearly indicate the top 10 games. But since, we are expected to display only top 10, the data is sorted and sliced. Additionally, we can also rank the games based on the total sales using a bar plot.

The second question asks to display the most popular genre per region. The result is shown using a map. The map is fit for this question as we can distinctly identify the top genre for each region based on the total sales. A barplot could also have been used here but having different barplots for different regions requires understanding each and every barplot. This is not feasible as the top genre for each barplot would have to be remembered. Instead using a map and a legend, we can effectively summarise all the information that the different barplots were trying to achieve. The map displays the top genre in each region and the legend can be used to identify which genre it is. The map can also be extended to a dataset where video games sales for each country is given.

The third question asks to display the top publisher for each genre game. The result is shown using a donut chart. The donut chart is fit for this question. Using the total sales for each publisher , we make a donut chart for each genre. The donut chart clearly indicates which publisher made the highest sales for which genre. The donut chart also indicates which publisher made the lowest sales for each genre. This can be used to rank the publishers based on their sales and pick which publisher the boss wants to work with.


List 3 reasons why D3 was helpful and improved your visualization (6 points)

1. D3 is helpful in summarizing data. The video games data consists of multiple sales for each game. It can be difficult to identify which game had the highest sale just by looking at the data. Additionally, it can also be difficult to compare the performance of a game with respect to other games. Both the problems are resolved by using D3, which can help a user draw multiple inferences from the graph.

2. D3 offers better interactivity. With D3, the user can easily set parameters according to his convenience and the graph displays the result accordingly. This helps the user to understand the data well and draw his or her observations from it. Rather than just reading a documentation on the effects of parameters on data, the user can see the effect taking place.

3. D3 makes it easier to visualize answers for specific questions. For instance, D3 can be used to easily identify which publisher performed the best for each genre without writing any additional script. It also makes it easier to visualize regions or areas where the data behaved differently.


List 3 reasons why D3 would not be the best tool for creating a visualization (6 points)

1. D3 would not be the best tool for summarizing textual data. D3 works well if there is numerical data that can be used to plot the graph. But in the case of textual data it may not be able to plot a graph that can summarize the data. Hence, some preprocessing may have to be done to extract numerical data from textual data and then D3 can be deployed.

2. D3 would not be the best tool when trying to compare a lot of data. For instance, when trying to find which top publisher for each genre, plotting sales for all publishers can make the data difficult to interpret and find the result. Hence, some kind of filtering interactivity has to be added to understand the data better.

3. D3 would not be the best tool when there are multiple parameters. For instance, if the dataset is huge and there are multiple parameters then plotting all parameters on a single graph would not lead to best visualization for the data. Infact the graph would be confusing. So, a filtering interactivity would have to be added to understand the data. But, if there are multiple parameters and if the user have to select multiple times for each iteration then the interactivity is tedious. Moreover, creating visualization for each and every parameter may be overwhelming and a lot of information to remember

« Accessible Data Visualization »

What kinds of users might find this dashboard accessible and who might have more difficulty? What additional actions might you take to make this dashboard more accessible to all audiences? Your response should refer to at least one of the readings and be about 1 paragraph.

Visually impaired users who are red-green color blind and other color blindness may find this dashboard accessible. Users with blue-yellow complete blindness may have difficulty with the dashboard since the first question uses a range bar. Since the range bar is not readable by the screen reader then I would change the range bar to either a dropdown list or another interactivity that can be read by the screen reader. Also, add skiplinks that would allow users to navigate to the graphs directly. Add keyboard navigation so that the dashboard can be navigated with only a keyboard.
